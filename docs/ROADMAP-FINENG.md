## 0. 能力对照（已有 vs 缺口）

| 金融工程常见能力 | GoldRush 现状 | 缺口 |
|------------------|---------------|------|
| 信号打分 | LLM + 量化双轨 | 因子 IC / 失效下线 |
| 样本外 / 校准 | 分桶、偏乐观修正、命中率 | Walk-forward、情景 Brier |
| 风控闸门 | 数据红档、双分冲突弃权 | 波动缩放、回撤刹车、新鲜度 SLA |
| 仓位映射 | 计划仓 % + 定投/卫星 | 日平滑、约束堆叠形式化 |
| 情景 / 相似日 | scenario similarity | 距离阈值、最少样本、概率校准图 |
| Regime | macro-regime | 首页强调「同阶段」校准 |
| 回测叙事 | tearsheet 基础 + Web 对错统计 | 纸面权益 / MaxDD 曲线 |
| 解释交付 | TL;DR、统一操作建议 | 驱动归因（利率/美元） |

**原则**：短板不在「再堆指标」，而在 **决策链路可审计、风险用数字表达、防过拟合、服务定投真实行为**。

## 1. 架构分层（新功能落点）

业界常见 Pipeline（QuantConnect LEAN / 投顾引擎简化）：

```
Signals（LLM / 量化 / flow / 相似日）
  → Risk & Gates（门禁 / 双分 / 波动 / 回撤 / 新鲜度）
    → Portfolio（仓位 / 定投层 / 再平衡）
      → Explain（MD / Web TL;DR / 操作建议）
```

| 规则 | 说明 |
|------|------|
| 新功能必须标明落在哪一层 | 避免面板乱长 |
| 任何 Signal **不得绕过 Gate** 直接改操作文案 | 与 `resolveOperationalAdvice` 一致 |
| Explain 层只翻译，不重新算仓 | 单一真相源 |

## 2. 金融工程：高价值方向

### 2.1 风险数字，而不只是方向分（优先）

定投用户更怕 **回撤与「加在半山腰」**，不是多两个因子。

| 输出 | 含义 | 落地想法 | 优先级 |
|------|------|----------|--------|
| 隐含波动 / ATR% | 近 20 日波动 | 仓位 × `vol_scalar`（波动高 → 降卫星仓） | P0 |
| 距 MA / 百分位 | 贵不贵 | 已有 percentile → 进仓位与 TL;DR | P0 |
| 建议最大回撤预算 | 此仓位下历史不利分位 | 用 `gold_prices` 滚动分位，**不靠 LLM** | P0 |
| 尾部风险指数 | 已有 `tail-risk.ts` | Web 一颗「尾部」色点，与可信度并列 | P1 |

**口诀**：分数说方向，风险数字说能扛多少。

### 2.2 信号研究卫生（防自嗨）

借鉴：Zipline 事件回测、学术 quant 的 walk-forward、AQR「因子动物园」警告。

| 做法 | 说明 | 优先级 |
|------|------|--------|
| Walk-forward 校准 | 只用 T 日前报告估规则，禁止全样本调阈值后报命中率 | P1 |
| 因子 IC / 衰减 | 对 `quantFactors` 做 5/10/20 日 Spearman IC；IC 低 → 降权或标「失效」 | P1 |
| 标签定义冻结 | **标签 = 交易日 T+5 收盘相对涨跌**；改定义必须 bump 版本号 | 规范 |
| 成本模型 | 金定投摩擦极低；关注 **跳空/隔夜**，勿用日内 tick 思维 | 规范 |

### 2.3 仓位：从档位表到约束堆叠

借鉴：风险平价缩水版、约束组合、投顾再平衡。

不必上二次规划，用线性规则即可：

```
target = base(score)
target = min(target, vol_cap)           # 波动上限
target = min(target, gate_cap)          # 门禁（已有）
target = min(target, dual_cap)          # 冲突（已有）
target = target * (1 - drawdown_brake)  # 近 60 日已亏再降
target = smooth(target, prev, maxΔ=10)  # 日度仓位别跳 30%
```

| 项 | 说明 | 优先级 |
|----|------|--------|
| 仓位日平滑 | 定投体验关键；避免昨日 80% 今日 30% | P0 |
| 波动缩放 | ATR 或 20 日收益标准差 | P0 |
| 回撤刹车 | 近窗最大回撤超阈 → 降卫星 | P1 |
| 再平衡带 | 偏离目标仓 >15% 才提示调整（投顾 IPS 思维） | P1 |

推荐 **下一包名称**：**风险约束仓位 v2**（平滑 + 波动 + 风险角标）。

### 2.4 情景与尾部：概率可校准

借鉴：概率预报 Brier score、reliability diagram。

| 项 | 说明 | 优先级 |
|----|------|--------|
| 情景概率校准 | 预报上行 30% 时，历史上行频率是否 ~30% | P1 |
| 相似日门槛 | 距离阈值 + 最少样本，防伪精确 | P1 |
| 情景 vs 总建议一致 | 情景操作句不得与 `resolveOperationalAdvice` 冲突 | 规范 |

### 2.5 Regime 真正进决策

借鉴：状态切换思想（不必上 HMM）。

| 项 | 说明 | 优先级 |
|----|------|--------|
| 同 regime 校准上 Web | 字段已有 `regimeHistoricalAccuracy` 等 | P0 |
| 过热段卫星上限 | `extended_rally` → 更低卫星仓 | P1 |
| 周更解释 | 用 regime 一句话解释「与上周差异」 | P1 |

### 2.6 宏观驱动归因（叙事工程）

借鉴：宏观 CTA 的 driver attribution（非因果承诺）。

日报极简块示例：

```
今日压力线索：实际利率 ↑ / DXY ↑ / …
历史相关方向（研究用，非因果）：…
```

优先级：P1（展示加强；字段部分已有）。

## 5. 明确不建议（反模式）

| 方向 | 原因 |
|------|------|
| 以「点位预测准确率」为对外 KPI | 近随机游走；应用区间 + 仓位 + 风险 |
| 按短期命中狂调 LLM/量化权重 | 与双分政策冲突；冲突应弃权 |
| 高频 / 套利类指标 | 与定投路径、延迟、信噪比不匹配 |
| 同时上 10 个新数据源 | 维护成本 > 信息增益；先 SLA 与 IC |
| Web 无限加面板 | 已做折叠；新能力优先进 TL;DR / 仓位约束 |
| HMM / 深度学习择时 | 样本短、过拟合；违背项目立意 |

## 6. 推荐落地顺序

```
1. 风险约束仓位 v2（平滑 + 波动）     ← 建议下一刀
2. 纸面 MaxDD / 简易权益
3. Regime 命中上 Web
4. 新鲜度 SLA
5. 因子 IC / walk-forward / 情景校准
```

验收口径（任意一包）：

- [ ] 有单元测试或可复现 CLI  
- [ ] 不绕过门禁 / 双分弃权  
- [ ] MD + Web 至少一处可见  
- [ ] 更新本文件「落地状态」列与 `OPTIMIZATION.md` 变更记录  
- [ ] 若改标签或阈值定义 → 写版本说明
