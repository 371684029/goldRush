# 用户价值改进：从讨论到产品

> 来源：定投用户「想尽量多提取有用信息」+「要不要追战争等热点」  
> 原则：**有用 = 能约束仓位或阅读重心**；热点无传导则不展示。

**相关**：`docs/DAILY-DELTA.md` · `docs/WEEKLY-REFLECT.md` · `docs/RELIABILITY.md` · `docs/ROADMAP-FINENG.md`

---

## 1. 产品目标（对齐讨论）

| 用户要什么 | 产品怎么给 | 不要什么 |
|------------|------------|----------|
| 多提取有用信息 | 固定「必看清单」+ 差分/对错/仓位/可信度 | 每天换一套故事 |
| 学会看报告 | 列表对错、周末反思、阅读优先级 | 点位准确率承诺 |
| 战争等热点 | **事件→黄金传导卡**（利率/美元/避险） | 热点刷屏、无传导也写长文 |

---

## 2. 已落地（可立即用）

| 能力 | 入口 |
|------|------|
| 较昨日差分 + 驱动 | analysis / Web 列表与首屏 |
| 列表双打分 + 对错/顺逆预测 | Web 历史每一行 |
| 周末错因反思 | `reflect --md`（周日 cron）→ 注入下次 analysis |
| 仓位% / 可信度 / 门禁 | 日报首屏 |

**推荐阅读顺序（产品内写死）**：① 仓位%与可信度 → ② 较昨日 → ③ 对错/反思 → ④ 双打分 → ⑤ 传导卡 → ⑥ 长文可选。

## 5. 热点关注 SOP（给用户，也约束产品）

每条新闻只问：

1. 会不会改 **实际利率/联储路径**？  
2. 会不会改 **美元**？  
3. 会不会改 **风险偏好（避险）**？  

三问都「不清楚/不会」→ 产品不展示、用户不据此加减仓。
