## 1. 数据真实性

> 核心问题：LLM 可以捏造数据。当前只有一个「信息可靠性五道防线」的概念设计，实际落地薄弱。

### 1.1 当前状态（2026-07-15）

| 数据 | 来源 | 可验证性 | 风险 / 状态 |
|------|------|----------|-------------|
| 伦敦金价 | Tavily+LLM **+** gold-api / 新浪 / Yahoo 锚定 | 中高 — 有直连瀑布 | ✅ 零值防线已落地；Yahoo 可能超时 |
| 上海金价 | Tavily+LLM | 中 — 仍偏搜索提取 | 锚定不足时可能缺失 |
| 美元指数 | Tavily+LLM **+** Yahoo/FRED/新浪 | 中 | FRED 现网常超时 |
| 10Y / TIPS | Tavily+LLM **+** FRED | 中 | 同上 |
| CFTC 持仓 | CFTC 官方 ZIP（本地解析） | 高 ✅ | 已解决 |
| GLD ETF 持仓 | Yahoo 份额估算（原 SPDR CSV 已失效） | 中 — 现网常失败 | ⚠️ 失败则中性 50，不编造 |
| VIX | LLM + 正则 | 中 | 时效仍弱 |
| 央行购金 | `pboc-grabber` 启发式 + 情绪 LLM | 低–中 | ⚠️ 公开页解析不稳 |
| 地缘风险 | LLM 定性 | 无 | 保持定性 |

**P0 已完成（数据真实性相关）**：数值 0 拒绝入库、forwardFill 净化、一致性短路、多源金价锚定、历史 LBMA 回落。  
**仍待**：搜索原文存档、GLD 稳定吨数源、PBOC 结构化源、数据质量 Web 面板。

### 1.2 问题详情

**问题 A：LLM 数据提取不可审计**

当前 `DataCollectorAgent` 把 Tavily 搜索结果扔给 LLM，LLM 提取出结构化数据后直接使用。如果金价提取错了，无法追溯是哪个搜索片段导致了错误。

```
搜索片段 → LLM 黑盒提取 → MarketData → 分析报告
                ↑
        这里发生了什么？无法审计
```

**问题 B：来源分级依赖 LLM 自评**

`sourceGrade: 'A' | 'B' | 'C'` 是 LLM 自己判断的。LLM 可能把某个来源标为 A 级但实际上不可靠。

**问题 C：缺少数值合理性检查**

如果 LLM 提取的金价是 $1,500（历史数据）而实际是 $2,358，没有机制检测这种"值是对的但时间是错的"的情况。

**问题 D：多源交叉验证形式化不足**

Validator 做了一些交叉验证，但主要在「有没有」的层面，缺少「有多一致」的量化指标。

### 1.3 优化方案

#### P0 — 搜索源文件存档（审计追溯）✅ 部分落地

每次 `analysis` 运行时，将原始 Tavily 搜索片段保存到 `docs/search-raw/YYYY-MM-DD.json`（实现见 `search-raw-archive.ts`）：

```json
{
  "timestamp": "2026-07-13T14:30:00Z",
  "queries": [
    {
      "query": "gold price XAUUSD spot today 2026",
      "results": [
        {"title": "...", "url": "...", "snippet": "Gold spot price $2,358.50 ..."}
      ]
    }
  ]
}
```

当价格异常时，可以回查原始片段确认 LLM 是否提取错误。存档文件设置为 30 天自动清理。

#### P0 — 数值异常检测

在 `Validator` 中增加本地规则检测：

| 规则 | 触发条件 | 动作 |
|------|----------|------|
| 价格跳变 | 日涨跌幅 > 3% | 标记 ⚠️，要求额外来源确认 |
| 价格停滞 | 连续 3 天价格完全不变 | 标记 ⚠️，可能数据未更新 |
| 偏离 Yahoo | 与 Yahoo GC=F 现价偏离 > 1% | 标记 ⚠️，优先使用 Yahoo 数据 |
| 汇率异常 | 伦敦金/上海金隐含汇率与真实汇率偏离 > 2% | 标记 ⚠️ |

#### P1 — 多源一致性量化

当前 `altPrices` 字段存在但未充分利用。改为：

```
伦敦金: $2,358.50
  来源1: Kitco     $2,358.50 (A级)
  来源2: Investing  $2,358.80 (B级) 偏差 0.01%
  来源3: Bloomberg  $2,358.20 (A级) 偏差 0.01%
  一致性: ✅ 3/3 一致 (偏差 < 0.1%)
```

```
伦敦金: $2,358.50
  来源1: Kitco     $2,358.50 (A级)
  来源2: 金十数据   $2,341.00 (B级) 偏差 0.74%
  一致性: ⚠️ 1/2 偏差较大，已采用 Kitco
```

#### P1 — 数据溯源标注

每个数据点在报告中标注来源链：

```
数据溯源:
  ┌ 伦敦金 $2,358.50
  │  ├─ Kitco (A级) — Tavily 搜索 → LLM 提取
  │  ├─ Investing (B级) — Tavily 搜索 → LLM 提取
  │  └─ Yahoo GC=F (交叉验证) — 本地 HTTP → 直接解析 ← 锚定参考
  ├ CFTC 净多 285,432 手
  │  └─ CFTC.gov ZIP (A级) — 本地解析 ← 100% 可追溯
  └ GLD 持仓 872.14 吨
     └─ SPDR CSV (A级) — 本地解析 ← 100% 可追溯
```

#### P2 — 数据真实性仪表盘

在 Web 端增加一个「数据质量」面板，显示：
- 每条数据的来源数量和一致性
- 数据新鲜度（采集时间）
- 异常标记（如有）

## 2. 数据覆盖度

> 核心问题：GoldRush 只用了一小部分可用数据。很多对黄金有直接影响的数据完全缺失。

### 2.1 覆盖度矩阵

```
完整度 = 已有数据 / 应有数据

黄金投资决策应覆盖的数据维度：
┌──────────────────────────────────────────────────────┐
│ 价格     ████████░░  伦敦现货/期货/上海金/ETF净值      │
│ 宏观     ██████░░░░  美元/美债/TIPS/Fed ← 缺实际利率   │
│ 持仓     ████░░░░░░  CFTC/ETF ← 缺COMEX OI           │
│ 成交量   █░░░░░░░░░  完全缺失                          │
│ 波动率   █░░░░░░░░░  仅VIX(股市) ← 缺黄金波动率GVZ     │
│ 跨市场   ░░░░░░░░░░  完全缺失(金银比/油金比/铜金比)     │
│ 资金流   █░░░░░░░░░  仅ETF ← 缺期货资金流/期权偏度     │
│ 国内     ██░░░░░░░░  仅上海金 ← 缺溢价/ETF份额/期货    │
│ 季节     ░░░░░░░░░░  完全缺失                          │
│ 央行     ██░░░░░░░░  仅PBOC文字 ← 缺定量数据            │
└──────────────────────────────────────────────────────┘
            约 30% 覆盖
```

### 2.2 缺失数据详解

#### P0 — COMEX 成交量 + 未平仓合约（零成本）

**当前**：Yahoo Finance `GC=F` 日线数据包含 `volume` 字段（成交量），但 `yahoo-gold-history.ts` 只提取了收盘价。

**改造**：增加 volume 提取，存入 `gold_prices` 或 `institutional_flows.comex_volume`。

**分析价值**：
- 放量上涨 = 趋势确认，缩量上涨 = 动能不足
- 放量下跌 = 恐慌抛售，缩量下跌 = 回调尾声
- OI 增加 + 价格上涨 = 新资金入场，趋势可靠
- OI 减少 + 价格上涨 = 空头平仓推动，涨势不稳

#### P0 — 实际利率（已有数据，未充分使用）

**当前**：TIPS 收益率已经在 `gold_prices.tips_yield` 和 `MarketData.usTreasury.tips` 中。

**问题**：没有在前端展示"实际利率 vs 金价"的关系。实际利率是金价最核心的定价锚之一。

**改造**：在 `price` 或 `analysis` 输出中增加一行：

```
实际利率 (10Y TIPS): 1.85%  → 对金价构成压力 (实际利率上升不利黄金)
```

#### P1 — 金银比（Gold/Silver Ratio）

**数据**：Yahoo `SI=F` 同样免费可用。

**分析价值**：
- 金银比 > 90 → 白银被低估，贵金属可能整体上涨
- 金银比 < 50 → 市场过热
- 金银比趋势 = 风险偏好指标（比高 = 避险，比低 = risk-on）

**改造**：`goldrush price --detail` 或 `flow` 命令增加金银比。

#### P1 — 上海金溢价

**数据**：已有伦敦金价和上海金价，差值 = 上海金溢价。

**分析价值**：
- 上海金溢价 > 0 → 中国需求强劲（利多）
- 上海金溢价 < 0 → 中国需求疲软

**改造**：`price` 输出增加一行：

```
上海金溢价: +$12.50/oz (中国需求偏强)
```

#### P1 — 国内黄金ETF份额 + 资金流

**数据**：东方财富 API（免费）。

**分析价值**：国内散户/机构资金流向。中国是全球最大黄金消费国，国内 ETF 份额变化有独立信号。

#### P2 — 黄金波动率（GVZ）

**数据**：CBOE Gold ETF Volatility Index。

**分析价值**：VIX 反映股市恐慌，GVZ 反映黄金市场恐慌。两者有时背离——当 GVZ 高但金价不跌时，可能酝酿突破。

#### P2 — 跨市场数据

| 数据 | 作用 |
|------|------|
| 原油 WTI | 通胀预期代理，与黄金同向 |
| 铜 | 工业需求，经济增长代理 |
| 比特币 | 替代"数字黄金"，竞争关系 |
| 美债 2Y-10Y 利差 | 衰退信号，利差倒挂利多黄金 |

#### P2 — 黄金季节性

黄金价格有明显的季节性规律（印度排灯节、中国春节、西方圣诞），可以作为辅助参考。

伦敦金: $2,358.50  🕐 14:30 CST (5分钟前)

GLD 持仓:    872.14 吨   ⏳ 数据截至昨日(T+1)

## 4. 预测可靠性

> 核心问题：LLM 输出一个确定数字（"68/100"），但用户不知道这个数字的可靠程度。

### 4.1 优化方案

#### P0 — 评分区间替代单点 ✅ 落地（可信度半宽）

```
当前: 综合评分: 68/100  偏多
改为: 综合评分: 63-73/100  偏多 🟢
      同分段 18 次分析，5日后 67% 上涨
      校准偏差: 偏乐观 8%（已下调）
```

实现：`reliability-card.ts` 按门禁/双分/样本推算半宽；MD/Web 展示 **low–high**。

#### P0 — 校准质量告示

样本不足时明确告知：

```
⚠️ 校准样本仅 4 次（需 ≥5），评分仅供参考
⚠️ 同分段样本 12 次（<20），准确率波动较大
✅ 同分段样本 25 次，5日涨概率 72%，历史偏差 3%
```

#### P1 — 四维度一致性

```
📊 维度一致性: 强一致 (3/4 偏多)
   技术 72 多 ✅  | 基本 65 多 ✅  | 情绪 68 多 ✅  | 基金 48 空 ❌
   → 基金面唱反调: 估值偏高建议减仓，但其他三维度偏多
   → 综合评分未受影响，但需关注基金面风险
```

当一致性低时（2/4 or 1/4），输出应明确标注"方向分歧"。

#### P2 — 回测追踪面板（Web）

Web 端增加一个图表页，展示过去 30 天的评分 vs 实际金价走势。

**部分落地（2026-07-16）**：非价格曲线图，而是 **历史预测对错统计面板**（命中率、分桶、近况明细）+ **当前仓位推荐**。见 `docs/POSITION-AND-TRACK.md`、`docs/goldrush-stats-latest.json`、`server.cjs`。完整「评分 vs 金价」时序图仍待做。

## 5. 交互简单易懂

> 核心问题：12 个命令、30+ 个选项、100+ 行输出。对不懂金融或不想读长文的人不友好。

### 5.1 优化方案

#### P0 — 默认仪表盘

`goldrush` 无参数直接输出 10 行关键信息：

```
🥇 GoldRush — 2026-07-13 14:30 CST
伦敦金 $2,358.50 | 上海金 ¥548.20 | 美元 104.32
─────────────────────────────────────────────
📊 研判 68/100 偏多 | 维持定投，急跌可加码
🏦 主力 CFTC偏多(78) ETF中性(62) 央行偏多(85)
⚠️ 背离: 无 | 一致性: 3/4 偏多
─────────────────────────────────────────────
analysis 查看完整报告 | flow 查看主力详情
```

#### P0 — 分数→操作映射（已部分实现，需统一）

```
 0-30: 🔴 暂停定投，等待企稳
30-45: 🟠 放慢节奏，不加仓
45-55: 🟡 维持基础定投
55-70: 🟢 正常定投，急跌加码
70-100:🔵 适度加码，高位不追
```

#### P1 — 分级输出

| 命令 | 输出行数 | 适用场景 |
|------|----------|----------|
| `goldrush` | ~10 | 日常一扫 |
| `goldrush analysis` | ~30 | 快速了解 |
| `goldrush analysis --full` | ~120 | 深度研究 |
| `goldrush analysis --md` | 完整 | 存档/分享 |

### 5 项具体改动

### 设计原则

- **重要信息高亮**：评分、方向、操作建议 → 最大、最亮、置顶
- **不重要信息折叠**：反驳摘要、尾部风险、评分构成、历史相似 → 默认折叠
- **简单易懂**：`68/100 偏多` → `维持定投，急跌可小幅加码`（人话翻译）
